前言：
由于找不到KMP算法对于next合理的推导解释，网上教程总有跳步骤或者没解释的问题，所以自己写的数学推导。本篇内容从零出发，只需要有一定递推思想，以及了解KMP算法的实现逻辑和next数组的基本定义即可看懂

KMP算法解释&next数组的定义

KMP算法

KMP算法的主要核心是模式串next数组值的计算，利用模式串对应的next数组值，避免了匹配不成功时不必要的回溯。所以我们要预先计算出next数组。

next数组的定义

网上next数组的定义有很多，这导致它们的初始化各有不同，并且存在PM表和next数组混淆的情况。这里我们采用最简单的一种定义方式，并且不采用大多数next数组的初始化方式（next[1] = 0 or next[1] = -1)，因为这些方式只是为了方便代码运行时不会陷入死循环所设计的，而不利于我们理解next数组的本质。下面给出next数组的定义：

**在模式匹配下，模式串的第i个字符不匹配的情况下，应该跳转到第next[i]个字符进行重新匹配。

由定义，我们可以得知当第1个字符不匹配时，应该将主串向后移一位，并重新开始匹配，则此时应该跳转到next[1] = 1，这是显然的，也是和网上解释不一样的地方，在实际代码中需要将其更改为任意一个不在next数组中的任意值即可，一般设为0或者-1。

递推发现

我们可以发现next数组的值和模式串的前后缀有一定联系，且前面next的值包含了前后缀的匹配信息。我们将next[i-1] 写出：

思路勘误
然后我们可以发现在next[i]中必定能找到一个$t_{1}t_{2}\dots t_{j-1}$，但是其对应的后缀匹配为：$t_{i-j+3}t_{i-j+2} \dots t_{i-2}t_{i-1}$ ，其并不对应我们已知的next[i] = j: $t_{i-j+2}t_{i-j+3} \dots t_{i-3}t_{i-2}$ 。故其不是我们要的递推关系。

所以我们在next[i]找到 $t_{1}t_{2}\dots t_{j} = t_{i-j+2}t_{i-j+3} \dots t_{i-2}t_{i-1}$ ，其和已知的 $t_{1}t_{2}\dots t_{j-1} = t_{i-j+2}t_{i-j+3} \dots t_{i-3}t_{i-2}$ 一一对应，除了最后一位$t_{j}$ 和 $t_{i-1}$ 。所以我们每次仅需要判断是否 $t_{j} = t_{i-1}$ 即可。

这也是为什么程序中判断语句 T.ch[i] == T.ch[j] ，我们每次的判断不用判断整个前后缀匹配，而是通过前面的next值即可推出。

注意，我们的 i 和程序中下标不同的原因是程序设计是从前向后计算，所以每次匹配成功会进行i++ 。而我们是通过 next[i] 从后向前推导，具体实现按程序逻辑即可。

$t_{j}$ 和 $t_{i-1}$ 情况分析

$t_{j} = t_{i-1}$ 成立

由于next[i] 和 next[i-1] = j 在对于前后缀匹配结果上差1，所以我们得出 next[i] = next[i-1] + 1 = j + 1 ， 所以程序当中，我们对于匹配不成功时，先做j = next[j] ，如果$t_{j} = t_{i-1}$ 成立，我们进行 j++ ，然后再赋值 next[i] 。

下面我们就应该考虑当 $t_{j} = t_{i-1}$ 不成立时，为什么程序会反复执行 j = next[j] 来进行匹配。

$t_{j} = t_{i-1}$ 不成立

事实上，我们只推导了 next[i] 和 next[i-1] 的情况，对于 i - 1 前面是否有可以匹配当前的后缀的前缀情况我们没有推导。而我们可以知道对于 next[i-1] = j ，其值为最大可以匹配的前缀情况，所以 i - 1 到 j 之间应该都没有可匹配的前缀。所以我们只需要判断 j 以前的情况，也就是推导 next[j] 是否有匹配的前缀。

所以我们再设：next[j] = k , 即有 $t_{1}t_{2}\dots t_{k-1} = t_{j-k+1}t_{j-k+2} \dots t_{j-2}t_{j-1}$ .

同理，next[i] 可以找到一个可能为：$t_{1}t_{2}\dots t_{k} = t_{i-k+1}t_{i-k+2} \dots t_{i-2}t_{i-1}$ .

我们可以看到当前后缀的下标是不一致的，无法一一匹配。但是由于我们已经匹配过 next[i-1] = j ,并且 匹配到的 j 的前后缀长度显然大于 k 。所以可以得知 $t_{1}t_{2}\dots t_{j-1} = t_{i-j+2}t_{i-j+3} \dots t_{i-3}t_{i-2}$ . 故联立两式。我们可以得出 ：

此时情况则和匹配 next[i] 的情况完全相同，我们只有 $t_{k} = t_{i-1}$ 是不清楚是否匹配的。故程序代码每次回重新赋值 j = next[j] , 然后判断 T.ch[i] == T.ch[j] 是否相等。

next初始值为 0 或 -1 的解释

根据上面的递推，如果next数组无法匹配到任何一个值，则会退回到 第一位，而此时程序会一直在第一位上做死循环，故设初始值为0避免这个情况。而设置 -1 则是因为实际代码中下标从0开始，索引依旧有哈希冲突，故设置为-1。
